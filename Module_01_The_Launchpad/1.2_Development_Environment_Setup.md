# 1.2 Development Environment Setup

## Wait... Where Do We Actually Write Python?

Imagine deciding to become a writer.

You learn grammar.

You learn vocabulary.

You learn how stories work.

Then someone asks:

> "Great! Which notebook are you using?"

And you realize you don't actually have one.

Programming has the same problem.

Before we can write code, we need a place to write it.

This place is called a **development environment**.

Fancy name.

Simple idea.

---

## What Is a Development Environment?

A development environment is simply the collection of tools that help us write, run, test, and debug programs.

Think of it as a programmer's workspace.

A carpenter needs:

* A workbench
* Tools
* Materials

A programmer needs:

* Python
* A code editor
* A terminal (sometimes)
* Patience

Especially patience.

---

## Step 1: Install Python

Python doesn't magically appear on your computer.

We need to install it.

Python can be downloaded from:

```text
https://python.org
```

During installation, Windows users should check:

```text
☑ Add Python to PATH
```

This small checkbox has saved countless hours of suffering.

And caused countless hours of suffering when forgotten.

---

## Step 2: Verify the Installation

Open a terminal and type:

```bash
python --version
```

or

```bash
python3 --version
```

If everything works, you'll see something like:

```text
Python 3.13.2
```

Congratulations.

Python is alive.

---

## Step 3: Choose a Place to Write Code

Technically, you could write Python in Notepad.

You could also cut vegetables using a ruler.

Neither is recommended.

Let's use proper tools.

---

## Option 1: VS Code (Recommended)

Visual Studio Code is one of the most popular code editors in the world.

Why people like it:

* Free
* Fast
* Beginner-friendly
* Supports almost every programming language
* Huge extension ecosystem

For most learners, VS Code is an excellent choice.

---

## Option 2: PyCharm

PyCharm is a dedicated Python IDE.

An IDE (Integrated Development Environment) is like a code editor with superpowers.

Advantages:

* Excellent Python support
* Smart suggestions
* Powerful debugging tools

Disadvantages:

* Heavier than VS Code
* Can feel overwhelming for complete beginners

---

## Option 3: Jupyter Notebooks

Jupyter Notebooks are popular among:

* Data scientists
* Researchers
* Students
* AI and machine learning practitioners

Instead of writing an entire program at once, you can execute code in small blocks.

This makes experimentation much easier.

---

## Which One Should You Use?

For this repository:

### Beginner Recommendation

Use:

```text
VS Code + Python Extension
```

It's simple, professional, and widely used.

As your projects grow, you'll naturally learn additional tools.

---

## Your First Python File

Create a file called:

```text
hello.py
```

Inside it, write:

```python
print("Hello, World!")
```

Save the file.

Run it.

If you see:

```text
Hello, World!
```

you have officially written and executed your first Python program.

The computer has obeyed your instructions.

Enjoy this feeling.

It won't last forever.

Soon you'll spend an hour searching for a missing parenthesis.

---

## First Principles Takeaway

Programming isn't just about learning a language.

It's also about building an environment where you can communicate with the computer efficiently.

Python is the language.

The editor is the notebook.

The development environment is the workspace.

Together, they form the foundation on which every project in this repository will be built.

---

## Debugger's Corner

You install Python.

You write code.

You run:

```bash
python hello.py
```

And the terminal responds:

```text
'python' is not recognized...
```

Welcome.

You have just discovered why programmers develop trust issues.

The solution is usually:

* Python wasn't added to PATH
* The installation failed
* The computer needs a restart

Don't worry.

Every programmer has met this problem at least once.
